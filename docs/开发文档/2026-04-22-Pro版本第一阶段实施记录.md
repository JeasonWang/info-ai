# Pro版本第一阶段实施记录

## 阶段目标

本阶段目标是先打稳 Pro 版本底座：完成 MySQL 建表、SQLite 数据迁移、`info-serve` Go 服务基础骨架，并为后续用户鉴权和管理后台拆分预留清晰边界。

## 已完成

- 保存完整 MySQL 建表语句：`info_aggregation/sql/mysql_schema_pro.sql`。
- 所有新建业务字段均保留中文注释，覆盖内容采集、事件聚合、用户鉴权、用户偏好、管理审计、采集监控和质量快照。
- 新增 SQLite 到 MySQL 迁移脚本：`info_aggregation/sql/mysql_migration_from_sqlite.py`。
- 迁移脚本支持首次建库、可重复清库重跑、日期格式归一化、历史脏外键过滤。
- 新增 `info-serve` Go 服务，当前包含：
- `GET /health`
- `POST /api/auth/register`
- 环境配置加载
- 密码哈希与校验
- 统一 JSON 响应
- 更新 `info_aggregation/deploy.env.example` 为 Pro MySQL 本地配置。
- 新增 `info-serve/deploy.env.example` 和 `info-serve/README.md`。
