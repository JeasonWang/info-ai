## 前端启动

进入前端目录：

```bash
cd /Users/jeasonwang/IdeaProjects/info-ai/info-max
```

安装依赖：

```bash
npm install
```

本地开发：

```bash
npm run dev
```

单测：

```bash
npm test -- src/views/__tests__/SettingsView.spec.ts
```

完整构建：

```bash
npm run build
```

## 后端启动

进入后端目录：

```bash
cd /Users/jeasonwang/IdeaProjects/info-ai/info_aggregation
```

创建虚拟环境：

```bash
python3 -m venv .venv
```

安装依赖：

```bash
.venv/bin/python -m pip install -r requirements.txt
```

启动后端：

```bash
.venv/bin/python main.py
```

## 当前推荐开发顺序

1. 先启动后端，确认接口可用。
2. 再启动前端，检查首页、详情页、设置页是否正常加载。
3. 每次修改科技采集器后，先跑对应渠道测试和 `test_tech_content_parser.py`。
4. 每次修改详情流水线或调度写回后，补跑两组后端组合回归。
5. 每次修改前端页面后，补跑对应组件测试和 `npm run build`。

## 当前重点链路

当前最关键的链路有两条：

1. 高完整度采集链路  
`BaseCrawler -> detail_pipeline -> scheduler -> Info / InfoAcquisitionLog`

2. 后台诊断链路  
`/api/infos -> 设置页采集质量看板`

3. Plus 科技语义链路  
`detail_pipeline -> tech_content_parser -> Info.tech_* -> /api/infos -> 详情页 / 设置页`

## 常见注意事项

- 优先使用项目内 `.venv`，不要混用系统 Python。
- Apple Silicon 下如果依赖安装异常，优先确认当前终端是否使用原生 ARM 环境。
- 采集器优化优先走“测试先失败，再补实现”的节奏，避免把反爬问题和内容质量问题混在一起。
- 设置页已经可以直接看详情状态、平均分、失败主因、策略分布和科技主题信息，后续调采集器时优先参考这个面板。
- 如果前端详情页没有看到科技关键词，优先检查 `/api/infos/{id}` 返回里 `tech_topic_type / tech_keywords` 是否为空，再回溯 `scheduler` 写回链路。
