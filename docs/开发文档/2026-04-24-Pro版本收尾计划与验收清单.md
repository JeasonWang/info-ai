# Pro 版本收尾计划与验收清单

更新时间：2026-04-24

## 收尾原则

Pro 版本已经完成架构拆分、MySQL 主库、用户鉴权、用户侧收藏/偏好/阅读历史、独立管理后台、管理动作闭环和渠道质量诊断。当前阶段不再继续扩展大功能，避免版本长期拖延。

本次收尾只保留三条主线：

1. 数据质量可修复：管理员不仅能看到低质量数据，还能对低完整详情进行重抓。
2. 服务链路可验收：`info_aggregation`、`info-serve`、`info-max`、`info-admin` 能同时启动，并完成核心接口冒烟。
3. 文档交付可追溯：保留 Pro 能力边界、验收账号、验收步骤和已知风险。

## 本次封版新增能力

### 低完整详情重抓

管理后台“数据质量”页新增“重抓低完整详情”动作。

执行链路：

```text
info-admin
  点击“重抓低完整详情”
info-serve
  POST /api/v1/admin/retry-low-quality-details?limit=20
  管理员鉴权 + 审计
info_aggregation
  POST /api/admin/retry-low-quality-details?limit=20
  按低质量信息分渠道重抓详情
  刷新语义字段
  保存质量快照
  重建事件流
```

低完整详情判定：

- 详情状态不是 `complete`。
- 详情评分低于 `80`。
- 详情正文长度低于 `120`。
- 正文为空。

批量控制：

- 默认每次最多处理 `20` 条。
- 接口上限为 `50` 条，避免一次性触发过重的爬虫任务。

## Pro 封版验收清单

### 用户端

- 打开 `http://127.0.0.1:5173/`。
- 首页可加载最新事件，最新数据日期应为当天或最近一次成功采集时间。
- 分类筛选、排序筛选和搜索可正常使用。
- 事件详情页正文不应只有列表摘要，应尽量展示详情正文。
- 登录用户可收藏事件、恢复首页筛选偏好、查看阅读历史。

### 管理后台

- 打开 `http://127.0.0.1:5174/`。
- 使用管理员账号登录：

```text
邮箱：admin@info-daren.local
密码：Admin123456
```

- 采集监控页可查看渠道健康、最新资讯时间、最新事件时间、内容均长、低完整数量和失败原因。
- 采集监控页可手动触发单渠道采集。
- 数据质量页可刷新质量、重抓低完整详情、归档低质量、归档重复标题。
- 审计日志可记录管理接口访问和操作。

### 服务接口

- `GET http://127.0.0.1:8000/` 返回采集服务状态。
- `GET http://127.0.0.1:18080/health` 返回业务服务状态。
- `GET http://127.0.0.1:18080/api/v1/events?page=1&page_size=1` 返回事件列表。
- `GET http://127.0.0.1:18080/api/v1/admin/channel-health` 需要管理员 token，并返回渠道诊断字段。
- `POST http://127.0.0.1:18080/api/v1/admin/retry-low-quality-details?limit=1` 需要管理员 token，并能触发低完整详情重抓。

## 已知风险

- 部分平台存在反爬和登录态限制，详情重抓不能保证所有渠道都达到 100% 完整，但现在已经具备可观测和可重试能力。
- 低质量归档是软删除，适合本地治理；后续生产化前需要增加二次确认、操作回滚和批处理进度展示。
- 当前一键启动和 Docker 部署仍属于开发环境能力，正式部署需要单独规划生产配置、安全域名、HTTPS 和日志采集。

## 后续版本建议

Pro 封版后，建议暂停横向功能扩张，进入短周期验收。下一版本再考虑：

- 订阅提醒和关注关键词。
- 采集任务队列化和异步进度。
- 管理后台趋势图与告警。
- 用户端个性化排序。
