# 2026-05 信息达人 3 周版本验收清单

## 产品定位验收

- [ ] 用户端第一屏能明确感知这是“AI 热点事件情报台”，不是普通热榜列表。
- [ ] 首页存在“可信事件 / 观察中”两类入口。
- [ ] 可信事件展示一句话判断、可信度、来源数、最新变化。
- [ ] 观察中事件明确说明为什么不能直接下结论。
- [ ] 详情页能回答：发生了什么、为什么重要、最新进展、风险提示、证据来源。

## 数据采集验收

- [ ] 后台数据质量页能看到 6 个核心信源：微博、今日头条、知乎、小红书、Reuters、36氪。
- [ ] 每个核心信源能看到 complete、partial、list_only、failed、pending 和可用率。
- [ ] 失败原因 Top N 可见。
- [ ] Reuters sitemap metadata 回填后不再大量停留在不可解释的 list_only。
- [ ] 401 / 403 / 404 等阻断型失败原因不会被吞成 `detail_unavailable`。

## 事件分析验收

- [ ] 事件重建前可执行预演，不写入数据。
- [ ] 预演报告包含候选组、单条组比例、多来源组比例、命中已有事件状态。
- [ ] 预演报告包含 `risk_groups`，可暴露疑似错合并或单源重复聚集。
- [ ] 无实体字段时合并保持保守，错合并优先级高于合并率。
- [ ] 无完整来源的事件默认进入 monitoring。

## 大模型验收

- [ ] LLM Prompt contract 明确要求结构化 JSON 输出。
- [ ] 一句话总结不能过短、不能半截话、不能照抄标题。
- [ ] LLM 输出不合格时记录失败日志并回退规则分析。
- [ ] LLM 失败不阻塞事件重建。
- [ ] 后台模型页能看到成功率、平均耗时、连续失败、最近失败原因、熔断状态。

## 后台治理验收

- [ ] 数据质量页能定位哪个核心信源采集差。
- [ ] 事件分析质量页能看到缺失分析、低置信度、fallback、展示拦截。
- [ ] 治理动作可用：刷新质量、重抓低完整详情、补偿分析弱来源、归档低质、归档重复。
- [ ] 详情任务页可按渠道和失败原因筛选。
- [ ] LLM 模型页可新增、编辑、选择聊天模型，并查看调用健康度。

## 本地联调流程

1. 启动四个服务：

```bash
./scripts/start-local.sh
```

2. 检查服务健康：

```bash
curl http://localhost:8000/health
curl http://localhost:8085/health
```

3. 执行核心质量审计：

```bash
cd info_aggregation
./.venv/bin/python tools/event_quality_audit.py --pretty
./.venv/bin/python tools/event_rebuild_preview.py --limit 300 --pretty
```

4. 小批量回填：

```bash
cd info_aggregation
./.venv/bin/python tools/backfill_reuters_sitemap_metadata.py --limit 100 --pretty
./.venv/bin/python tools/backfill_event_display_quality.py --limit 300 --pretty
```

5. 页面验收：

- 后台：`http://localhost:5174/data-quality/report`
- 事件分析质量：`http://localhost:5174/data-quality/event-analysis`
- LLM 模型：`http://localhost:5174/llm-models`
- 用户端首页：`http://localhost:5175`

6. 关闭服务：

```bash
./scripts/stop-local.sh
```
