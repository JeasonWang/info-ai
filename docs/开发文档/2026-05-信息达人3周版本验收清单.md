# 2026-05 信息达人 3 周版本验收清单

## 产品定位验收

- [x] 用户端第一屏能明确感知这是“AI 热点事件情报台”，不是普通热榜列表。
- [x] 首页存在“可信事件 / 观察中”两类入口。
- [x] 可信事件展示一句话判断、可信度、来源数、最新变化。
- [x] 观察中事件明确说明为什么不能直接下结论。
- [x] 详情页能回答：发生了什么、为什么重要、最新进展、风险提示、证据来源。

## 数据采集验收

- [x] 后台数据质量页能看到 6 个核心信源：微博、今日头条、知乎、小红书、Reuters、36氪。
- [x] 每个核心信源能看到 complete、partial、list_only、failed、pending 和可用率。
- [x] 失败原因 Top N 可见。
- [x] Reuters sitemap metadata 回填后不再大量停留在不可解释的 list_only。
- [x] 401 / 403 / 404 等阻断型失败原因不会被吞成 `detail_unavailable`。

## 事件分析验收

- [x] 事件重建前可执行预演，不写入数据。
- [x] 预演报告包含候选组、单条组比例、多来源组比例、命中已有事件状态。
- [x] 预演报告包含 `risk_groups`，可暴露疑似错合并或单源重复聚集。
- [x] 无实体字段时合并保持保守，错合并优先级高于合并率。
- [x] 无完整来源的事件默认进入 monitoring。
- [x] 完整可用的单源事件不再被固定判为低置信度。
- [x] 历史低置信完整来源事件可批量标记并重分析。
- [x] 事件分析质量报告能展示风险事件的主要问题和下一步动作。

## 大模型验收

- [x] LLM Prompt contract 明确要求结构化 JSON 输出。
- [x] 一句话总结不能过短、不能半截话、不能照抄标题。
- [x] LLM 输出不合格时记录失败日志并回退规则分析。
- [x] LLM 失败不阻塞事件重建。
- [x] 后台模型页能看到成功率、平均耗时、连续失败、最近失败原因、熔断状态。

## 后台治理验收

- [x] 数据质量页能定位哪个核心信源采集差。
- [x] 事件分析质量页能看到缺失分析、低置信度、fallback、展示拦截。
- [x] 治理动作可用：刷新质量、重抓低完整详情、补偿分析弱来源、归档低质、归档重复。
- [x] 详情任务页可按渠道和失败原因筛选。
- [x] LLM 模型页可新增、编辑、选择聊天模型，并查看调用健康度。

## 本地联调流程

1. 启动四个服务：

```bash
./scripts/start-local.sh
```

2. 检查服务健康：

```bash
curl http://localhost:8000/health
curl http://localhost:8085/health
```

3. 执行只读验收快照：

```bash
./scripts/acceptance-smoke.sh
```

输出目录：`logs/local/acceptance/<时间戳>/summary.md`。

该脚本会记录：

- 采集服务和业务服务健康。
- 核心信源采集质量、主要问题、下一步动作。
- 事件分析风险、展示拦截样本和治理动作建议。
- 用户端事件列表抽样、疑似摘要问题数量。
- 事件详情页结构是否具备情报判断、证据链、争议提示、多源视角、历史关联。

4. 执行核心质量审计：

```bash
cd info_aggregation
./.venv/bin/python tools/event_quality_audit.py --pretty
./.venv/bin/python tools/event_rebuild_preview.py --limit 300 --pretty
```

5. 小批量回填：

```bash
cd info_aggregation
./.venv/bin/python tools/backfill_reuters_sitemap_metadata.py --limit 100 --pretty
./.venv/bin/python tools/backfill_event_display_quality.py --limit 300 --pretty
```

6. 小批量重分析补偿：

```bash
curl -sS -X POST 'http://localhost:8000/api/admin/mark-low-confidence-event-analysis-stale?limit=100'
curl -sS -X POST 'http://localhost:8000/api/admin/rebuild-stale-event-analysis?limit=100'
./scripts/acceptance-smoke.sh
```

该流程用于修复“来源已完整，但历史分析仍为旧低置信结果”的事件；执行后重点观察风险事件、低置信度、模型回退和首页疑似摘要问题。

7. 真实联调触发：

```bash
for ch in weibo toutiao zhihu xiaohongshu reuters 36kr; do
  curl -sS -X POST "http://localhost:8000/api/crawl/trigger?channel_code=$ch"
done
curl -sS -X POST 'http://localhost:8000/api/admin/event-analysis-detail-jobs?limit=20'
curl -sS 'http://localhost:8000/api/admin/channel-quality-report?sample_limit=3'
curl -sS 'http://localhost:8000/api/admin/event-analysis-quality-report?limit=5'
```

8. 页面验收：

- 后台：`http://localhost:5174/data-quality/report`
- 事件分析质量：`http://localhost:5174/data-quality/event-analysis`
- LLM 模型：`http://localhost:5174/llm-models`
- 用户端首页：`http://localhost:5175`

9. 关闭服务：

```bash
./scripts/stop-local.sh
```
