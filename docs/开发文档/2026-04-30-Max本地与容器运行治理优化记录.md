# Max 本地与容器运行治理优化记录

日期：2026-04-30

## 目标

Max 阶段明确两套运行方式：

- 开发阶段：本地直接运行四个服务，使用本机 MySQL `127.0.0.1:3306`。
- 上线阶段：使用根目录 `docker-compose.yml` 容器化部署，MySQL 使用容器内服务。

本次优化重点是减少配置分叉、日志误读、慢渠道采集超时和模拟数据干扰。

## 服务边界

| 服务 | 开发启动 | 上线启动 | 职责 |
| --- | --- | --- | --- |
| `info_aggregation` | `scripts/start-local.sh` 调用 Python venv | `docker-compose.yml` | 采集、清洗、详情补全、事件构建、质量快照 |
| `info-serve` | `go run ./cmd/server` | `docker-compose.yml` | 用户端 API、管理 API、鉴权、审计 |
| `info-max` | Vite dev server `5173` | Nginx 容器 `80` | 用户端 |
| `info-admin` | Vite dev server `5174` | Nginx 容器 `8081` | 管理端 |

## 本地运行规则

本地启动：

```bash
./scripts/start-local.sh
```

本地关闭：

```bash
./scripts/stop-local.sh
```

本地迁移：

```bash
./scripts/migrate-local.sh
```

本地脚本不启动 MySQL、不启动 Docker，不再把 MySQL 可用性判断作为启动前置拦截。默认参数：

```text
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=root1234
MYSQL_DB=info-max
```

如果本机账号不同，通过环境变量覆盖即可。

## 日志规则

开发阶段统一查看：

```text
logs/local/info_aggregation.log
logs/local/info-serve.log
logs/local/info-max.log
logs/local/info-admin.log
```

`start-local.sh` 会给 Python 采集服务显式传入 `LOG_DIR=logs/local`，避免同时出现多份 `info_aggregation.log` 后看错日志。

容器阶段优先看 Docker 日志：

```bash
docker compose logs -f info-aggregation
docker compose logs -f info-serve
```

采集服务同时保留 `/app/data/logs` 作为文件日志目录，便于需要时挂载排查。

## 配置规则

新增根目录 `.env.example` 作为统一配置模板。

开发阶段：

- 可以使用 shell 环境变量覆盖本地脚本默认值。
- 知乎 Cookie/ZSE 可以放根目录 `.env`，采集代码会直接读取，不需要 shell `source`。

上线阶段：

- `deploy.sh` 首次创建 `.env` 时生成随机 MySQL 密码、会话密钥和管理员初始密码。
- `docker-compose.yml` 向 `info-aggregation` 传入 `ZHIHU_COOKIE`、`ZHIHU_ZSE_93`、`ZHIHU_ZSE_96`。
- 未配置知乎凭据时服务仍可运行，但知乎高质量详情会降级。

## 模拟数据规则

Max 阶段默认不再启动时灌入模拟数据。

如需演示数据，显式配置：

```text
ENABLE_SEED_DATA=true
```

这样真实采集验收不会被 seed 数据干扰。

## 手动采集规则

管理后台点击“立即采集”后，`info-serve` 调用 `info_aggregation`，采集服务立即返回：

```json
{
  "status": "accepted",
  "trigger_type": "manual"
}
```

真实采集在后台执行，并写入 `crawl_run_log`。这样知乎、头条等慢渠道不会因为 HTTP 请求超时被误判为失败。

## 部署入口规则

正式容器部署只使用根目录：

```bash
./deploy.sh prod local-build
```

或：

```bash
./deploy.sh prod image-tar
```

`info_aggregation/docker-compose.prod.yml` 保留为历史单服务调试文件，已标注不作为 Max 正式部署入口。
