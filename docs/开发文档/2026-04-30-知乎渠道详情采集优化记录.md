# 知乎渠道详情采集优化记录

日期：2026-04-30

## 背景

Max 版本要求采集完整、有价值、可分析的数据，不能只保存标题或列表摘要。知乎渠道当前真实线上匿名访问存在限制：

- `https://www.zhihu.com/api/v3/feed/topstory/hot-lists/total?limit=20` 返回 401。
- `https://www.zhihu.com/hot` 匿名访问返回 403 或跳转登录页。
- Chromium 渲染匿名 `https://www.zhihu.com/hot` 会进入登录页，无法获得热榜内容。

因此知乎渠道和今日头条不同：今日头条可匿名抓热榜并渲染详情，知乎稳定采集需要登录态 Cookie。

## 本次优化

### 1. 支持登录态 Cookie

新增 `ZHIHU_COOKIE` 环境变量支持，知乎 API、网页请求、浏览器渲染链路都会带上 Cookie。

示例：

```bash
export ZHIHU_COOKIE='z_c0=...; d_c0=...; _xsrf=...'
```

同时支持从项目根目录或 `info_aggregation` 目录的 `.env` 文件中自动读取：

```text
ZHIHU_COOKIE=z_c0=...; d_c0=...; _xsrf=...
```

Cookie 会按原始文本读取，不需要通过 shell `source .env`，避免 Cookie 中的 `&`、`|` 等字符被 shell 错误解析。

同时支持知乎请求头中的 ZSE 配置：

```text
ZHIHU_ZSE_93=101_3_3.0
ZHIHU_ZSE_96=2.0_xxx
```

注意：ZSE 与 Cookie 属于登录态/风控相关配置，开发记录只记录配置项，不保存真实值。

### 2. 热搜入口升级为 `hot_search`

原 `topstory/hot-lists/total` 在当前登录态下仍容易返回 401。根据真实浏览器请求，新增入口：

```text
https://www.zhihu.com/api/v4/search/hot_search
```

本次真实响应字段为：

```text
hot_search_queries
```

根因定位：之前代码只兼容 `top_search.words`、`data`、`hot_search`，没有兼容 `hot_search_queries`，导致接口已经返回 200 但代码解析结果为 0 条。

### 3. 热词二次搜索定位详情源

知乎热搜入口给出的主要是热词，不等于完整详情。为避免只保存标题，本次新增二次搜索：

```text
https://www.zhihu.com/api/v4/search_v3?t=general&q={query}&correction=1&offset=0&limit=5
```

处理策略：

- 对每个热词调用 `search_v3`。
- 优先提取 `hot_timing.object.description.object` 中的问题对象。
- 兼容 `search_result.object` 中的文章对象。
- 将问题映射为 `https://www.zhihu.com/question/{id}`。
- 将文章映射为 `https://zhuanlan.zhihu.com/p/{id}`。
- 提取 `title`、`description`、`content` 并清理 HTML 标签、实体和空白。

这样列表项已经具备可分析正文，不再只依赖标题。

### 4. 搜索内嵌正文作为详情候选

新增 `search_embedded` 详情策略：当 `search_v3` 已返回足够完整的问题描述或文章正文时，直接作为详情候选进入详情质量评分。

如果 `search_embedded` 已经通过完整性校验，则提前返回，不再继续请求容易 403 的知乎问题网页，减少噪声和耗时。

### 5. 回答 API 扩展字段

知乎问题详情 API 增加 `include` 参数：

```text
data[*].content,voteup_count,comment_count,created_time,updated_time,author
```

这样可以拿到完整回答正文、点赞数和评论数，为后续价值评分做基础。

### 6. 优先选择高价值回答

回答详情不再简单取前 3 条，而是按：

1. `voteup_count`
2. 正文长度

排序，跳过过短回答，最多合并 3 条高价值回答，并去重。

### 7. 浏览器渲染兜底

当 API 不可用但页面可以渲染时，新增 `rendered_question` 详情策略：

- 使用 Playwright Chromium 渲染问题页。
- 过滤登录墙、评论、互动按钮、发布时间、短噪声。
- 只保留足够长、可分析的回答正文。

### 8. 登录墙识别

如果渲染结果出现：

- 验证码登录
- 扫码登录
- 登录/注册

则直接判为空内容，不把登录页文本写入详情。

## 真实验证

当前本地环境已从 `.env` 读取到 `ZHIHU_COOKIE`、`ZHIHU_ZSE_93`、`ZHIHU_ZSE_96`。

```text
ZHIHU_COOKIE_LENGTH = 1887
ZHIHU_ZSE_93 = configured
ZHIHU_ZSE_96 = configured
ZH_ITEM_COUNT = 20
```

真实采集前 3 条结果：

| 序号 | 标题 | 来源 | 热度 | 详情策略 | 状态 | 分数 | 长度 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 微信朋友圈改版，文字描述从配图右侧移至上方，并新增时间轴浏览功能，你发现了吗？你喜欢这个新样式吗？ | `question/2032722050247124724` | 505 万 | `search_embedded` | complete | 100 | 404 |
| 2 | 国安部披露某境外组织资助「躺平网红」相关话题 | `question/2032406989402289190` | 442 万 | `search_embedded` | complete | 100 | 652 |


覆盖能力：

- Cookie 注入请求头。
- `.env` 自动读取 Cookie。
- CookieJar 注入浏览器上下文。
- `hot_search_queries` 当前真实结构解析。
- 热搜词二次搜索到问题/文章。
- 搜索内嵌正文作为详情。
- 回答 API 正文解析。
- 多回答去重合并。
- 高赞、高信息量回答优先。
- 页面渲染兜底。
- 登录墙内容拒绝入库。

## 后续建议

1. 后台增加渠道凭据状态提示，知乎未配置 Cookie/ZSE 时显示“需要登录态”。
2. 增加知乎回答价值评分：点赞数、评论数、回答长度、作者权重、时间新鲜度。
3. 为 `search_v3` 增加请求缓存，避免同一轮采集重复查询相同热词。
