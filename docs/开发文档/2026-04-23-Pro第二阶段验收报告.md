# Pro 第二阶段验收报告

更新时间：2026-04-23

## 本阶段目标

本阶段将 Pro 版本从“后台可触发动作”继续推进到“可运营、可诊断、可验收”：

- 管理后台展示渠道健康评分。
- 用户端具备邮箱注册、登录、退出和登录状态展示。
- 提供本地一键启动脚本，降低多服务联调成本。

## 已完成能力

### 采集运营

- `info-serve` 新增 `GET /api/v1/admin/channel-health`。
- 渠道健康指标包括：
  - 最近运行次数。
  - 采集成功率。
  - 详情完整率。
  - 失败次数。
  - 健康分。
  - 健康等级：`healthy`、`warning`、`risk`。
- `info-admin` 采集监控页新增“渠道健康”面板。

### 用户侧登录

- `info-max` 新增 `/login` 页面。
- 支持邮箱注册、登录、退出。
- 未登录仍可浏览首页、详情、时间线。
- 登录后顶部展示当前邮箱。
- 收藏同步仍作为下一阶段任务，当前本地收藏能力保留。

### 本地启动

新增脚本：

```bash
./scripts/start-local.sh
```

脚本会启动：

- `info_aggregation`：`8000`
- `info-serve`：`18080`
- `info-max`：`5173`
- `info-admin`：`5174`

## 验收账号

管理后台：

```text
邮箱：admin@info-daren.local
密码：Admin123456
```

用户端可使用任意未注册邮箱进行本地注册。

## 验收清单

- 打开 `http://127.0.0.1:5174/`，登录管理后台。
- 进入采集监控页，查看渠道健康分、成功率、详情完整率。
- 点击“立即采集 央视体育网”，确认操作返回成功提示。
- 打开 `http://127.0.0.1:5173/`，确认首页仍可匿名浏览。
- 打开 `http://127.0.0.1:5173/login`，注册并登录普通用户。
- 登录后确认顶部展示当前邮箱，可退出。

## 已知风险

- 用户收藏、阅读历史、筛选偏好仍未迁移到 MySQL 账号体系。
- 渠道健康评分目前基于已有采集日志做聚合，后续可引入按时间窗口统计。
- 一键启动脚本适合本地开发，不是生产部署方案。
