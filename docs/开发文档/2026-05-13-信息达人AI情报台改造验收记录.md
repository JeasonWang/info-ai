# 2026-05-13 信息达人 AI 情报台改造验收记录

## 本轮目标

按 `docs/superpowers/plans/2026-05-13-info-daren-ai-intelligence-upgrade.md` 开始第一批快速改造，优先建立数据质量基线、低价值内容识别、事件展示质量门槛和规则分析防劣化能力。

## 已完成改造

1. 新增质量审计工具：
   - `info_aggregation/tools/event_quality_audit.py`
   - 支持输出渠道采集质量、事件摘要质量、分析运行质量和 LLM 调用质量。
2. 新增低价值内容识别：
   - 识别 `hot/new`、互动计数、热榜分类、纯表情、纯编号、Reuters sitemap 元数据等不能作为事实依据的文本。
3. 放宽标题清洗与 crawler 标题截断：
   - `clean_title()` 从 40 字放宽到 200 字。
   - 各 crawler 的 `title[:40]` / `word[:40]` 调整为 200 字上限，避免英文标题半截话。
4. 新增事件展示质量门槛：
   - `display_quality_score`
   - `display_quality_level`
   - `display_quality_reason`
   - 弱社交热度信号进入 `monitoring`，非社交事实来源仍可进入 `active`。
5. 规则分析防低价值摘要：
   - 规则分析不再把 `互动：点赞xx` 作为一句话摘要。
   - 弱来源输出“热度线索 / 缺少完整事实来源”的谨慎文案。
6. LLM 事件分析审计增强：
   - 事件分析 LLM 调用会把 request payload、response content、response payload 写入 `llm_call_log`。
   - 后续管理端可以基于日志抽检坏摘要对应的 prompt、模型返回和来源上下文。
7. `info-serve` API 输出增强：
   - 事件列表和详情透出 `display_quality_score`、`display_quality_level`、`display_quality_reason`。
   - `info-mvp` 类型定义已补充这些字段，后续可用于卡片可信度展示。
8. 新增数据库迁移：
   - `info_aggregation/sql/migration_v1.5.0_event_display_quality.sql`
   - 本地 MySQL `info-max.event` 已执行迁移。
9. 新增历史事件展示质量回填：
   - `info_aggregation/tools/backfill_event_display_quality.py`
   - 可对 `active/monitoring/low_quality` 事件批量重算展示质量，并按结果分流到 `active` 或 `monitoring`。
10. 新增 LLM 事件分析轻量重试：
   - 新增 `EVENT_ANALYSIS_LLM_RETRY_TIMES`，默认每次事件分析最多尝试 2 次。
   - 瞬时 timeout 或临时无效返回会先重试；最终仍失败时才记录失败日志并回退规则分析。
   - 避免一次临时网络/模型抖动直接让高价值事件退回规则摘要。
11. 管理端事件展示质量可解释治理：
   - `event-analysis-quality-report` 增加展示质量分流统计：追踪事件、可展示、已拦截、展示可用率、状态分布、等级分布、拦截原因 Top。
   - 管理端“事件分析质量”页增加展示质量拦截样本，直接显示拦截状态、展示分、等级、原因和更新时间。
   - 运营可以直接判断事件是“分析质量差”还是“来源质量弱导致不展示”。
12. 跨源事件聚合补强：
   - 在事件构建阶段增加同分类相似锚点合并。
   - 对没有 `core_entity/tech_entities/tech_keywords` 的社会热点，使用标题/正文共享线索进行保守合并。
   - 覆盖“广西公交车坠翻致3死5伤”和“广西梧州公交事故救援进展”这类不同标题写法但同一事件的跨源聚合。
13. Reuters 官方 sitemap 质量补强：
   - sitemap 采集增加栏目、发布时间、图片说明、股票代码、Reuters news code 等官方结构化上下文。
   - 详情补采支持识别已入库的 Reuters sitemap metadata，不再依赖 `_reuters_source` 临时字段。
   - 新增 `info_aggregation/tools/backfill_reuters_sitemap_metadata.py`，可无网络、无节流回填历史 Reuters metadata 质量字段。
14. 用户端展示质量表达：
   - `EventList` 卡片显示“高可信 / 可信 / 观察 / 质量分”。
   - 事件详情头部显示展示质量等级和主要原因，让用户知道当前结论的可信程度。
15. 微博/小红书事实源校验：
   - 社交平台即使抓到较长正文，也必须包含官方、通报、警方、媒体、事故、数据、报告等事实锚点，才可能进入 `active`。
   - 纯网友讨论、穿搭、互动热度、粉丝关注等内容会进入 `monitoring`，避免用户端首页被“有热度但无事实”的线索污染。

| 今日头条 | 热点详情阈值从文章口径调整为热点短事实口径，补充媒体、价格、消费、婚姻、政策等价值词 |
