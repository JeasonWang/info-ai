# Max 采集服务全渠道详情优化最终记录

日期：2026-05-02

## 优化目标

本轮围绕“采集完整、有价值、可分析的真实线上数据”推进，对 `info_aggregation` 采集服务做全渠道详情质量升级。重点不是把所有短摘要都标成成功，而是让每个渠道都有差异化完整性标准、详情策略名、质量分数和失败原因。

## 核心改造

### 1. 渠道差异化详情标准

文件：

- `info_aggregation/services/detail_pipeline.py`

新增 `ChannelDetailProfile`，按渠道类型定义不同详情标准：

- 热搜话题：微博、今日头条。
- 问答：知乎。
- 社交笔记：小红书。
- 技术/商业文章：CSDN、掘金、博客园、36氪、路透。
- 财经指标：东方财富。
- 体育新闻：央视体育、新浪体育。

结果：

- 文章渠道不再把几十字摘要判为完整正文。
- 热搜渠道允许较短但有上下文的事件描述进入 partial/complete。
- 财经指标必须包含指标值和市场解释。
- 体育新闻必须包含比赛、赛事、球队、球员、比分等信息密度。

### 2. 调度与详情闭环修复

文件：

- `info_aggregation/scheduler/__init__.py`
- `info_aggregation/api/__init__.py`
- `info_aggregation/crawlers/registry.py`
- `info_aggregation/services/detail_job_worker.py`

完成内容：

- 管理后台配置的渠道采集间隔通过 `CrawlTask.next_run_at` 真正驱动调度。
- 采集完成后推进下一次 `next_run_at`。
- `_search_content` 嵌入详情完整时同步写入语义字段和采集日志。
- 同渠道爬虫加锁，避免手动采集、定时采集、详情补偿并发复用同一个爬虫实例。

## 后续建议

1. 为微博、头条、小红书配置可维护的 Cookie / 代理池 / 浏览器上下文池。
2. 为 Reuters 确定稳定授权源或替代国际新闻源。
3. 在管理后台增加“渠道真实样本验收”页面，直接展示每个渠道最近 10 条的正文、策略、分数和失败原因。
4. 为 partial 内容建立二次补偿优先级，优先补正文短但热度高的内容。

## 当前验收结论

本轮已完成全渠道详情采集底座优化。系统现在可以按渠道差异化判断正文完整性，并对每条详情保留策略、分数、长度和失败原因。相比优化前，标题扩写和短摘要不再被轻易判为完整正文，高价值文章和问答渠道可以稳定保留长正文。

## 2026-05-03 小红书与 36 氪二次优化记录

本轮针对当前数据库评分较低的 `xiaohongshu` 和 `36kr` 继续优化，全部基于真实线上采样，不使用模拟数据判断效果。

### 小红书 `xiaohongshu`

问题：

1. 详情完整度波动大，部分笔记只有标题或短标签。
2. 动态页在未带 Cookie 或渲染兜底时容易返回登录壳页。
3. 原正文只拼接标题和描述，缺少标签、点赞、收藏、评论等价值上下文。

优化：

1. 新增 `XHS_COOKIE` 读取，支持环境变量和项目根目录 `.env`。
2. 请求头自动注入 `XHS_COOKIE`。
3. `INITIAL_STATE` 正文提取增加标签与互动数据。
4. Playwright 兜底改为只在轻量策略完全失败时触发，避免短笔记被渲染流程拖慢。
5. 渲染兜底从 `networkidle` 改为 `domcontentloaded`，并用 `JSON.stringify/JSON.parse` 安全读取 `window.__INITIAL_STATE__`。
6. 详情质量规则增加小红书登录壳页识别，避免登录页进入正文。

真实复测结果：

```text
XHS_COOKIE: 已加载，长度 803
采集列表: 20 条
样本:
- 炸了！东契奇再次前往德国治疗！ complete / xhs_initial_state / score=100 / len=271
- 2026程序员现状 complete / xhs_initial_state / score=92 / len=131
- DeepSeek-V4才发完没几天，竞品就到了？ complete / xhs_initial_state / score=100 / len=440
- 为什么我的Claudecode加kimi2.5这么笨 complete / xhs_initial_state / score=100 / len=471
```

结论：小红书 Cookie 生效后，真实正文完整度明显提升，且正文包含标签和互动信息。剩余 partial 多数是笔记本身较短，不应强行扩写成完整正文。

### 36 氪 `36kr`

问题：

1. 裸域 `https://36kr.com/p/...` 当前容易返回验证码安全挑战页。
2. 旧热榜 API payload 已不适配真实接口，导致入口不稳定。
3. 网页兜底没有解析 `window.initialState.articleDetail.articleDetailData.data.widgetContent`，只能拿到标题、作者和日期。

优化：

1. 热榜 API payload 改为 `partner_id + timestamp + param(siteId/platformId/pageSize)` 结构。
2. 热榜列表解析兼容 `templateMaterial.widgetTitle/itemId/statRead/statPraise/statCollect/statComment`。
3. 列表 URL 改为 `https://www.36kr.com/p/{itemId}`，避开当前裸域验证码页。
4. 网页兜底新增 `window.initialState` 解析，优先提取 `widgetTitle/summary/widgetContent`。
5. 验证码页增加显式识别：`sec_sdk_build/captchaOptions/验证后继续访问/滑块验证`。
6. 主入口失败时保留 Google News 索引兜底，只恢复真实线索，不把索引标题伪装为完整正文。

真实复测结果：

```text
采集列表: 17 条
样本:
- 续集没翻车 ，但这次“女魔头”也顶不住了 complete / web_fallback / score=100 / len=4465
- 茶颜悦色 , 装不下去了 complete / web_fallback / score=100 / len=2172
- 一边跌价，一边天价，水果市场为何走向两级？ complete / web_fallback / score=100 / len=4163
- 软件业大洗牌：要么AI增收10%，要么利润40% complete / web_fallback / score=100 / len=4495
- 苹果官方App误打包了Claude.md complete / web_fallback / score=100 / len=2121
```

结论：36 氪已从“入口不稳定、短正文”恢复为官方热榜入口 + 多数文章完整正文。个别文章只暴露短摘要时仍标记为 partial，不伪装完整。

## 2026-05-03 Reuters、头条、知乎二次优化记录

### Reuters `reuters`

问题：

1. Reuters 官方首页、World 页和文章页在当前网络环境下返回 401。
2. Google News RSS 只能提供标题级线索，缺少 Reuters 官方 URL 的稳定性和发布时间。
3. sitemap 元数据如果继续触发详情 API，会产生大量 401 噪声和无效等待。

优化：

1. 新增 Reuters 官方新闻 sitemap 入口：`https://www.reuters.com/arc/outboundfeeds/news-sitemap/?outputType=xml`。
2. 只保留 `world/business/markets/technology/legal` 等高价值分类，过滤纯体育类线索。
3. sitemap 结果记录官方 Reuters URL、发布时间和来源策略 `_reuters_source=news_sitemap`。
4. sitemap 元数据直接进入 `news_sitemap_metadata` partial，不再继续请求被拦截的详情页。
5. Google News RSS 保留为最后兜底。

真实复测结果：

```text
采集列表: 15 条
样本:
- Taiwan president defiant as begins Eswatini trip; China calls him a 'rat'
  partial / news_sitemap_metadata / score=59 / len=295
- Trump says US could restart Iran strikes 'if they misbehave'
  partial / news_sitemap_metadata / score=59 / len=292
- Germany says US troop drawdown should spur Europe
  partial / news_sitemap_metadata / score=61 / len=326
```

结论：Reuters 在当前无代理/无授权环境下仍无法抓取全文，但已从“无数据或 Google 标题摘要”提升为“官方 sitemap 线索 + 官方 URL + 发布时间”。这些内容只能作为 partial 元数据，不伪装完整新闻正文。

### 今日头条 `toutiao`

问题：

真实样本中，部分头条事件正文已经达到 300-700 字，但因 `value_markers` 过窄，被误判为 `low_channel_value_density`。

优化：

1. 扩展头条事件型价值关键词：交通、出行、游客、比赛、小组赛、选手、冠军、夺冠、去世、影响、提醒等。
2. 保持短内容阈值不变，仍防止标题或热榜标签冒充正文。

真实复测结果：

```text
- 假期第二天 全国路网维持高位运行 complete / rendered_page / score=100 / len=619
- 张雪得知夺冠后瞬间弹起挥拳嘶吼 complete / rendered_page / score=100 / len=423
- 主持人熹菲抗癌10年去世 年仅37岁 complete / rendered_page / score=100 / len=385
- 吴宜泽首次晋级斯诺克世锦赛决赛 complete / rendered_page / score=100 / len=182
```

结论：头条真实详情提取链路基本可用，本轮主要修正评分误伤。仍偏短的热点会继续保持 partial。

### 知乎 `zhihu`

问题：

知乎热搜二次搜索嵌入内容经常是 150-700 字的事件/问答摘要，其中财经、社会事件类内容不一定包含“回答/观点/讨论”等旧关键词，导致误判为低价值。

优化：

1. 知乎 profile 的完整阈值从 220 调整为 150，更符合热搜嵌入详情的真实长度。
2. 扩展知乎价值关键词：如何、解读、报告、数据、同比、分析、影响、事件等。
3. 仍保留 20 字以下失败和弱相关判断，避免标题级内容进入完整正文。

真实复测结果：

```text
- 五粮液 2026年Q1净利同比大增82.57% complete / search_embedded / score=100 / len=667
- 孙杨被曝疑似挤占他人名额 complete / search_embedded / score=100 / len=736
- 导游称大熊猫花花是残疾 complete / search_embedded / score=100 / len=383
```

结论：知乎评分更贴近真实热搜问答数据。标题级或 100 字左右的短线索仍保留为 partial。
