# Pro版本MySQL建表说明

本文档对应 `info_aggregation/sql/mysql_schema_pro.sql`，用于保存 Pro 版本完整建表语句和字段注释。

## 设计原则

- 所有表使用 `InnoDB`、`utf8mb4`、`utf8mb4_unicode_ci`，保证事务、外键和中文内容兼容。
- 所有业务字段必须带中文 `COMMENT`，方便后续管理后台、数据排查和交接维护。
- 现有内容与事件表继续承载用户侧展示，新增用户、鉴权、审计、采集监控、质量快照表支撑 Pro 能力。
- 数据库名使用用户提供的 `info-max`，由于包含连接符，SQL 中统一使用反引号。

## 表分组

- 内容采集表：`category`、`channel`、`info`、`info_acquisition_log`。
- 事件聚合表：`event`、`event_item_link`、`event_timeline_entry`、`event_summary_snapshot`。
- `event.event_key` 保存事件稳定键，用于事件重建时复用同一事件身份，保护用户收藏、阅读历史等业务关系。
- 用户鉴权表：`user_account`、`user_session`。
- 用户增强表：`user_favorite_event`、`user_follow_keyword`、`user_preference`、`user_read_history`。
- 管理审计表：`admin_audit_log`。
- 采集监控表：`crawl_task`、`crawl_run_log`、`crawl_health_snapshot`、`data_quality_snapshot`。

## 第一阶段边界

第一阶段先保证数据底座和服务边界成立。用户注册、登录、管理后台权限会由 `info-serve` 逐步接管；`info_aggregation` 暂时仍保留原 FastAPI 管理接口，等 `info-admin` 和 `info-serve` 完成后再迁移。
