# 详情页体验统一优化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将事件详情页和资讯详情页统一打磨为移动端优先、正文和结论优先的阅读体验。

**Architecture:** 保持现有 Vue 页面组件，不新增全局状态。`EventDetailView.vue` 重排首屏重点信息；`InfoDetailView.vue` 将采集诊断信息折叠；`styles.css` 补充紧凑详情页样式；现有测试更新为新体验预期。

**Tech Stack:** Vue 3 `<script setup>`、Vue Router、Vitest、Vue Test Utils、CSS 响应式布局。

---

## Tasks

### Task 1: 事件详情页首屏降噪

- [ ] 将 `detail-hero__support` 改成一个主结论卡和两条紧凑辅助信息。
- [ ] quickfacts 只保留更新时间和来源数量，不重复显示时间线条数。
- [ ] 更新 `EventDetailView.spec.ts`，确认仍展示重点结论、最新进展、为什么重要，但不再依赖大网格堆叠。
- [ ] 运行 `npm test -- --run src/views/__tests__/EventDetailView.spec.ts`。

### Task 2: 资讯详情页阅读优先

- [ ] 首屏保留标题、频道、来源、时间、原始来源按钮。
- [ ] 移除首屏采集策略、评分、科技诊断卡片。
- [ ] 将采集状态、策略、评分、事实、科技主题、实体、关键词放进默认折叠的“采集信息”区。
- [ ] 更新 `InfoDetailView.spec.ts`，确认诊断信息默认折叠，正文仍去重展示。
- [ ] 运行 `npm test -- --run src/views/__tests__/InfoDetailView.spec.ts`。

### Task 3: 样式与全量验证

- [ ] 在 `styles.css` 增加详情页紧凑辅助信息、折叠采集信息和阅读页样式。
- [ ] 运行 `npm test -- --run`。
- [ ] 运行 `npm run build`。
