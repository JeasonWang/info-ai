# info-mvp 用户登录体验修复记录

## 问题

1. 用户注册成功后没有进入登录态，需要再次手动登录。
2. 首页右上角登录后的字体图标依赖 `uniicons` 字形，部分端显示模糊且语义不清晰。
3. 前端注册表单提示密码至少 6 位，但后端注册规则要求至少 8 位。

## 修复

1. `info-serve` 注册接口改为注册成功后直接创建会话，返回 `{ token, user }`，与登录接口保持一致。
2. `info-mvp` 注册接口类型改为 `LoginResult`，注册成功即可写入 token 和用户信息。
3. 首页右上角从三个纯图标按钮改为“收藏 / 足迹 / 用户字母头像”，提升 H5、小程序和 PC 多端清晰度。
4. 前端密码校验和占位提示改为至少 8 位，与后端规则一致。

## 验收

1. 新用户注册后应直接回到首页，并显示登录后的右上角操作区。
2. 刷新页面后登录态应保持，收藏、足迹入口可见。
3. 右上角按钮不再依赖模糊字体图标，移动端显示应保持清晰。
