## 1. 项目最终目的

`信息达人` 的长期目标，是成为一个面向大众用户的 `事件理解器`：

- 从社交平台、新闻网站、财经网站、科技网站、体育网站等来源采集热点内容
- 将同一热点在不同平台的内容进行聚类与聚合
- 用 `一句话看懂`、`事件时间线`、`多来源归纳`、`背景补充` 帮助用户快速理解热点
- 让用户不再需要在多个 App 或网站之间频繁切换

一句话定义：

`信息达人是一个面向大众热点用户的事件理解产品，用事件聚合替代平台切换，用摘要和时间线帮助用户更快看懂全网热点。`

## 4. 当前架构原则

项目当前采用的核心架构思想：

1. `采集层和理解层分离`
爬虫采集与事件理解不写死在同一层，便于后续扩来源、换算法、接 API 数据。

2. `以事件为中心`
原始内容是证据材料，用户真正消费的是 `event` 对象，而不是离散的信息条目。

3. `平滑演进`
MVP 阶段保留已有 `info` 体系作为内容层，在其上新增 `event` 体系，而不是一次性推翻重写。

4. `处理链可扩展`
事件聚类、摘要生成、时间线整理、热度计算等能力，后续都可以继续独立升级。

5. `前后端解耦`
前端逐步从“直接消费 info 列表”迁移为“消费事件 API”，便于继续扩展 Web、H5、小程序、App。
