# 首页无限滚动 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 移除首页底部分页页码，改为滚动到底部自动追加下一页热点事件。

**Architecture:** `HomeView.vue` 负责分页状态、追加数据、IntersectionObserver 哨兵和搜索/频道重置；`EventList.vue` 只负责渲染列表和底部加载状态，不再引用 `PaginationBar`；测试通过 mock `IntersectionObserver` 主动触发加载更多。

**Tech Stack:** Vue 3 `<script setup>`、Vitest、Vue Test Utils、IntersectionObserver、现有分页 API。

---

## File Structure

- Modify: `info-max/src/views/HomeView.vue`
  - 新增 `loadingMore`、`loadMoreError`、`hasMore`、底部哨兵 ref。
  - `loadEvents(page, mode)` 支持 replace 与 append。
  - 增加 `IntersectionObserver` 自动加载下一页。

- Modify: `info-max/src/components/EventList.vue`
  - 移除 `PaginationBar`。
  - 增加底部状态展示和重试事件。

- Modify: `info-max/src/views/__tests__/HomeView.spec.ts`
  - mock `IntersectionObserver`，验证滚动哨兵触发下一页并追加列表。

- Modify: `info-max/src/components/__tests__/EventList.spec.ts`
  - 验证不再渲染分页页码，并展示底部状态。

- Modify: `info-max/src/styles.css`
  - 增加无限滚动底部状态样式。

## Tasks

- [ ] 更新 `EventList.vue` props：移除 `page/pageSize` 使用，新增 `hasMore`、`loadingMore`、`loadMoreError`。

- [ ] 更新 `EventList.spec.ts`：

### Task 2: HomeView 支持 replace/append 加载

- [ ] 将 `loadEvents(page = 1)` 改为 `loadEvents(page = 1, mode: 'replace' | 'append' = 'replace')`。
- [ ] replace 模式用于初次加载、搜索、切换频道。
- [ ] append 模式用于加载下一页，成功后将旧 `items` 与新 `items` 合并。
- [ ] 新增 `hasMore = computed(() => eventPage.value.items.length < eventPage.value.total)`。
- [ ] 新增 `loadMoreEvents()`：
  - 如果 `loading/loadingMore/!hasMore`，直接返回。
  - 请求 `eventPage.value.page + 1`，使用 append 模式。
- [ ] 更新模板传参给 `EventList`。

### Task 3: IntersectionObserver 自动触发下一页
