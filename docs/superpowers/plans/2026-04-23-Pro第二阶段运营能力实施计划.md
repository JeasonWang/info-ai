# Pro 第二阶段运营能力实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 Pro 版本从“后台可操作”升级为“可运营、可诊断、可验收”的产品化状态。

**Architecture:** 第一阶段先在 `info-serve` 汇总 MySQL 中的采集日志，输出渠道健康评分；`info-admin` 用该接口展示渠道健康、成功率、详情完整率和失败概览。随后补用户侧登录态入口和本地验收脚本，避免一次性做过深的个性化推荐。

**Tech Stack:** Go `net/http`、MySQL、Vue3 + TypeScript + Vite、Shell 脚本。

---

## 任务 1：采集监控与渠道健康评分

- [ ] 新增 `ChannelHealth` 类型和 `ListChannelHealth` 服务方法。
- [ ] 从 `channel`、`crawl_task`、`crawl_run_log` 聚合最近 20 次采集结果，计算成功率、详情完整率、健康分和风险等级。
- [ ] 新增 `GET /api/v1/admin/channel-health` 接口，继续走管理员鉴权与审计。
- [ ] `info-admin` 采集监控页展示渠道健康列表。

## 任务 2：用户侧登录后基础能力

- [ ] `info-max` 增加登录状态服务，支持注册、登录、退出、当前用户查询。
- [ ] 首页和详情页保留匿名浏览，登录后显示用户状态。
- [ ] 收藏能力先复用当前本地收藏，文档中标注下一步迁移到 MySQL 同步。

## 任务 3：一键启动与验收文档

- [ ] 新增本地启动脚本，统一启动四个服务。
- [ ] 新增 Pro 第二阶段验收报告，记录服务地址、账号、验证命令和已知风险。
- [ ] 全量测试、构建、重启服务、提交。
