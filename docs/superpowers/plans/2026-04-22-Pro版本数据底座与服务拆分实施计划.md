# Pro版本数据底座与服务拆分实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立 Pro 版本的 MySQL 数据底座、SQLite 到 MySQL 的迁移能力，并创建 `info-serve` Go 服务的业务 API 与鉴权基础骨架。

**Architecture:** `info_aggregation` 继续负责采集和数据治理，统一写入 MySQL；`info-serve` 负责用户侧与管理侧 API、鉴权、业务编排；`info-max` 继续作为移动端优先的用户前端；后续新增 `info-admin` 作为 PC 端管理与监控后台。

**Tech Stack:** Python 3、SQLAlchemy、PyMySQL、MySQL 8、Go 1.24、Gin、go-sql-driver/mysql、bcrypt、pytest、go test。
