# 信息达人 AI 情报台产品说明

> 适用版本：v1.0.0 上线候选版  
> 更新时间：2026-05-15

## 产品定位

信息达人是面向热点事件的 AI 情报台。它不做普通热榜聚合，也不只是把新闻做摘要，而是把多渠道信息整理成可验证、可追踪、可决策的事件情报。

产品要回答的问题是：

1. 这件事到底发生了什么？
2. 哪些信息是真的、有来源？
3. 来源是否完整、可信、可追溯？
4. 事件处于发酵、扩散、反转、降温还是长期议题阶段？
5. 用户是否值得继续关注？

## 用户角色

| 角色 | 核心需求 | 当前能力 |
|---|---|---|
| 普通用户 | 快速看懂可信热点，少被噪声带偏 | 首页可信事件、观察中事件、详情页情报摘要 |
| 深度用户 | 查看证据链、来源差异、风险提示 | 事件详情页展示来源、证据、多源视角和风险提示 |
| 运营人员 | 发现采集和分析问题，触发治理动作 | 管理后台数据质量、事件分析质量、详情补偿、弱来源治理 |
| 管理员 | 管理渠道、凭证、模型、审计 | 分类渠道管理、凭证管理、LLM 配置、审计日志 |

## 当前产品范围

### 用户端

当前唯一维护的用户端是 `info-mvp`，支持 H5 和微信小程序构建。

核心页面：

- 首页事件流：展示可信事件和重点观察事件。
- 事件详情页：展示一句话判断、发生了什么、为什么重要、最新进展、风险提示、证据链、多源视角、历史关联。
- 登录注册：支持账号登录、注册和登录态保持。
- 收藏和阅读历史：支持用户留存和回看。

### 管理端

管理后台是 `info-admin`，通过 `info-serve` 统一鉴权和审计，不直接连接数据库。

核心页面：

- 数据质量：核心信源完整率、可用率、失败原因、治理建议。
- 详情任务：按渠道和失败原因查看、重试、取消详情补偿任务。
- 事件分析质量：风险事件、低置信度、模型回退、展示拦截、弱来源治理。
- LLM 模型：配置 OpenAI Compatible 模型，查看调用健康度和熔断状态。
- 分类渠道：维护分类、渠道、采集间隔、启停状态。
- 凭证管理：维护微博、小红书、知乎等渠道 Cookie 和扩展凭证。

## 当前不做

1. 不恢复 `info-max`，用户端统一走 `info-mvp`。
2. 不把弱社交来源放宽进首页可信事件流。
3. 不把管理后台做成复杂 BI，先保证可定位、可治理、可验收。
4. 不默认开启 LLM，系统必须在规则分析下稳定运行。
5. 不在当前版本加入订阅推送、个性化推荐、社区评论等增长功能。

## 上线判断

当前版本已经具备上线候选条件：

- 四个服务可本地启动和生产容器化部署。
- 数据库完整 schema 和初始化数据已收敛到 `info_aggregation/sql/`。
- 用户端能表达“可信事件 / 观察中”的产品差异。
- 管理后台能定位采集质量、事件分析质量、展示质量和模型健康。
- 低置信度和模型回退已经归零，剩余风险集中在来源质量。

当前剩余产品风险：

- 微博和小红书受登录态、反爬、动态页面影响，可用率仍偏低。
- 今日头条短热点已经改善，但仍需要持续抽样。
- 来源质量治理应继续通过 pending 队列、Cookie 有效性验证和二跳事实源召回提升，不能通过放松首页门槛解决。
